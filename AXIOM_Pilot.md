# AXIOM — спецификация пилота (v2)

> **AXIOM — твой личный ИИ-SDR.** Автоматизирует твой процесс продаж риелторам в Telegram: написал → договорился на Zoom → встреча в календарь → напоминание → дожим.
> **Фаза 1 — под себя (догфудинг). Фаза 2 — упаковка в продукт/SaaS.**

---

## 1. Границы пилота (зафиксировано 2026-06-16)

| Параметр | Решение |
|---|---|
| Кто пользователь | ты сам (личный ИИ-SDR) |
| Аудитория аутрича | **риелторы — коллеги по рынку**, тёплый B2B |
| Источник базы | **твоя база контактов/телефонов** (импорт), НЕ парсинг |
| Канал | **только Telegram** |
| Объём | 10–20 сообщений/день |
| CRM | **своя БД + NocoDB** (Bitrix → фаза 2) |
| Хранение | SQLite |

**Отложено в фазу 2+:** MAX, WhatsApp, Avito · Bitrix24 · парсинг чатов · AI-досье физлиц · мульти-агентный «нейроотдел» · свой дашборд · нетворкинг/инвайтинг/мероприятия.

---

## 2. Пайплайн пилота (6 шагов Олега Аверса, минус Bitrix)

```
[Твоя база риелторов]
        │ импорт (телефоны/@username)
        ▼
1. ИНИЦИИРОВАНИЕ ПЕРЕПИСКИ  ── персональное первое сообщение в TG
        ▼
2. ДИАЛОГ → ДОГОВОРЁННОСТЬ О ZOOM  ── ИИ-агент ведёт, предлагает слот
        ▼
3. ВНЕСЕНИЕ ВСТРЕЧИ  ── Google Calendar + Zoom-ссылка
        ▼
4. ПОДТВЕРЖДЕНИЕ  ── автонапоминание за 2–3 ч до встречи
        ▼
5. (Bitrix — фаза 2)  ── протокол + стадия. Сейчас: запись в свою БД
        ▼
6. ДОЖИМ / ПОВТОР  ── не ответил/не пришёл → follow-up, новый слот
```

## 3. Архитектура

```
              ┌──────────────────────────────────────┐
              │            AXIOM (VPS Linux)          │
              │                                        │
Telegram ◄─MTProto─►  [Userbot]  Telethon              │
(1-3 акк, прокси)        │                             │
                         ▼                             │
              [Оркестратор + антибан-лимиты]            │
                 │        │            │               │
                 ▼        ▼            ▼               │
          [AI-агент]  [Планировщик]  [Хранилище]       │
          Claude API   APScheduler    SQLite           │
          диалог,      напоминания,   contacts/        │
          договор о     дожим          messages/deals   │
          встрече         │              │              │
                          ▼              │              │
              [Google Calendar + Zoom API]              │
                          │              │              │
              [NocoDB] ── таблицы/фильтры/канбан ◄───────┘
              └──────────────────────────────────────┘
```

### Компоненты
- **Userbot (Telethon)** — импорт контактов, отправка, чтение ответов. 1 аккаунт на старте, до 3 при росте; каждый — свой прокси.
- **AI-агент (Claude)** — ведёт диалог с риелтором, цель = договориться о Zoom; распознаёт «да/нет/позже/возражение», предлагает слот.
- **Планировщик (APScheduler)** — напоминание за 2–3 ч, дожим (не ответил 24/48 ч; не пришёл → новый слот).
- **Хранилище (SQLite)** — contacts / messages / deals.
- **Интеграции** — Google Calendar (событие + инвайт), Zoom (ссылка на конференцию).
- **NocoDB** — интерфейс: таблицы с фильтром/группировкой/сортировкой, канбан по стадиям сделки.

## 4. Модель данных

**contacts** (риелтор из твоей базы):
```
id, source(import), phone, tg_user_id, username, name, city, agency,
notes, status, tags, created_at, updated_at
# status: new → messaged → in_dialog → meeting_set → met → won/lost/nurture
```

**messages** (диалог):
```
id, contact_id, direction(in/out), text, intent, ts
# intent: positive / objection / later / not_interested / question
```

**deals** (встречи/воронка):
```
id, contact_id, stage, zoom_link, meeting_at, calendar_event_id,
reminder_sent, next_action_at, outcome, notes
```

*В фазе 2: + AI-досье, граф связей (edges), парсинг, Postgres, Bitrix-синк.*

## 6. Антибан (риск ниже, чем при холоде, но соблюдаем)

- 1 аккаунт = 1 прокси (мобильный/резидентный РФ).
- Импорт контактов порциями, ловим FloodWait.
- 10–20 первых сообщений/день, паузы рандомизированы, «человеческий» темп.
- Stop-list: «не интересно/отстаньте» → больше не пишем.
- Тёплая база (коллеги) → жалоб мало, но прогрев аккаунта 3–7 дней всё равно делаем.

## 9. Метрики успеха пилота
- Отправлено первых сообщений: 10–20/день, **0 банов**
- Ответы: цель ≥ 20–30% (тёплая база — должно быть высоко)
- Назначено Zoom-встреч: ≥ несколько
- Дошли до встречи (подтверждение сработало): измеряем
- Полный цикл «сообщение → встреча в календаре» работает без ручных действий
