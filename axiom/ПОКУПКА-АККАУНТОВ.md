# Чек-лист: покупка TG-аккаунтов и ввод в строй

Написано после разбора 16.07.2026, когда из 38 аккаунтов слетело 16 — **целыми лотами**.
Причина у всех одна: `сессия не авторизована (разлогинен)`. Не бан, не связь — нас **выкинули**.

---

## 0. Почему аккаунты умирают лотами (главное, что надо понять)

Ты покупаешь **сессию**, но не **номер**. Номер остаётся у продавца.

Пока на аккаунте нет **твоего** облачного пароля (2FA), продавец в любой момент:
1. вводит номер в Telegram → получает SMS-код (телефон-то его);
2. заходит и жмёт «завершить все другие сеансы»;
3. твоя сессия мертва, а войти заново нечем — кода у тебя нет.

Это называется **реклейм**, и делается пачкой по всему проданному лоту. Ровно это
случилось с лотами `+1 740…`, `+1 716…` и `+998 95…`.

**Вывод: единственное, что реально защищает — свой 2FA, поставленный сразу.**

### Честная граница 2FA

2FA — не броня, а **замедлитель**. Владелец номера может запросить сброс пароля, и
Telegram отдаст ему аккаунт **через 7 дней**. Итог: вместо мгновенного увода — неделя
форы и заметный след (придёт уведомление). Полная защита = 2FA + recovery-email,
но email требует ввода кода из почты руками.

## 1. Перед покупкой (защита от фишинга и мусора)

- **Только через маркет lzt с гарантией/эскроу.** Ни шагу в ЛС: «давай напрямую, дешевле» —
  это увод денег. Оплата мимо маркета = гарантии нет.
- **Проверь домен в адресной строке.** Фишинг живёт на похожих доменах (`lolz`, `lzt` с
  лишней буквой, другая зона). Заходи из своей закладки, а не по ссылке из чата/письма.
- **Никогда не вводи данные СВОЕГО Telegram** на сторонних сайтах и «чекерах». Ни номер,
  ни код, ни 2FA. Легальный чекер аккаунта никогда не просит твой личный аккаунт.
- **Смотри поля лота:**
  - `twoFA` — **должен быть в комплекте.** Лоты без 2FA-пароля — риск (см. п.0).
  - почта/email в комплекте — большой плюс.
  - `spamblock` — должен быть чистый.
  - дата регистрации — чем старше, тем живучее.
  - есть ли доступ к номеру/SMS — если да, аккаунт восстановим.
- **Запиши срок гарантии лота.** Это твоё единственное окно на возврат при реклейме.

---

## 2. Сразу после покупки — первые 15 минут (пока идёт гарантия!)

Порядок важен. Не меняй его.

1. **Сначала прокси, потом вход.** Никогда не логинься с домашнего IP и не с чужой страны.
   Прокси = страна номера. Резкая смена гео = мгновенный бан.
2. **Положи файлы** в `АККАУНТЫ ТГ/sessions/` (`.session` + `.json` рядом).
3. **Импортируй и проверь:**
   ```
   python -m channels.import_session       # лот → БД (без сети)
   python -m channels.proxy_check --ids N  # прокси живой?
   python -m channels.session_check --ids N # сессия живая?
   ```
4. **НЕМЕДЛЕННО ставь свою 2FA** — это гонка с продавцом:
   ```
   python -m channels.twofa --dry          # посмотреть кандидатов
   python -m channels.twofa --ids N        # поставить
   ```
   Пароль сохраняется в `accounts.tg_2fa` **до** установки — потерять его нельзя.
5. **Если `2FA уже стоит, но пароль не наш`** — аккаунт под контролем продавца, ты его не
   защитишь. Либо требуй пароль у продавца, либо **сразу в диспут, пока идёт гарантия**.
6. **Если сессия слетела в гарантийный срок** — открывай спор. «Разлогинен продавцом» —
   типовой повод для возврата. Это деньги, а не потери.

> **Срок проверки = срок гарантии.** Проверять «на следующей неделе» бессмысленно:
> гарантия кончится, и реклейм станет твоей проблемой, а не продавца.

---

## 3. Как не убить аккаунты самому

- **Один аккаунт = один постоянный прокси.** Не переставляй IP туда-сюда — для Telegram
  это «аккаунт угнали».
- **Страна прокси = страна номера.** `+998` через немецкий IP — красный флаг.
- **Никогда с домашнего IP.** Спалишь связку «свой личный ↔ боевые» — заберут всех разом.
- **Прогрев перед рассылкой.** Свежий аккаунт, сразу пишущий 50 незнакомцам, живёт день.
  Колонка «ПРОГРЕВ» в пульте — не украшение.
- **Соблюдай лимит/день.** Колонка «ЛИМИТ/ДЕНЬ». Больше — не быстрее, а мертвее.
- **Не вступай в 20 чатов за раз** одним аккаунтом. Раскидывай по агентам и по времени.
- **Не шли одинаковый текст** с разных аккаунтов — это ловится на раз. Для того и варианты
  в кампании (`{привет|здравствуйте}`).
- **Родные номера (`🏠 protected`) не трогает автоматика.** Не снимай этот флаг с личного.

---

## 4. Что проверять регулярно

```
python -m channels.proxy_check      # все прокси (пачками по 8, не врёт)
python -m channels.session_check    # живость сессий
python -m channels.twofa --dry      # кто остался без нашей 2FA
```

Три сигнала в пульте — **разные вещи**, не путать:

| Сигнал | Что значит | Чинится |
|---|---|---|
| `proxy_alive` | жив ли канал связи | сменить прокси |
| `session_alive` | не выкинули ли нас | релогин (нужен SMS-код!) |
| `spam_status` | ограничения у живого аккаунта | отлежаться, снизить темп |

`session_alive=NULL` — это **«не смогли проверить»**, а не «мёртв». Не выключай аккаунт по NULL.
