# Оргструктура: визуальная схема + авто-импорт ресурсов

**Дата:** 2026-07-20
**Проект:** AXIOM — личный ИИ-SDR
**Статус:** Specification

## 1. Цель

Переработать раздел «Оргструктура» в веб-пульте AXIOM: заменить карточное представление на визуальную org-диаграмму (древовидную схему с линиями-соединителями), добавить автоматический импорт аккаунтов из раздела «Ресурсы» как виртуальных сотрудников, сохранить табличный список как вторую вкладку.

## 2. Что меняется

### 2.1 Вкладка «🗺️ Схема» (была «🏛️ Схема»)

Вместо карточек-списков отделов — **настоящая org-диаграмма**:

```
         ┌──────────────┐
         │  Василий CEO  │
         └──────┬───────┘
    ┌───────────┼───────────┬──────────┐
 ┌──┴───┐   ┌──┴───┐   ┌──┴───┐   ┌──┴───┐
 │Продажи│   │Нетворк│   │Тёплые │   │Админ │
 │       │   │       │   │       │   │      │
 │👤 Анна│   │🤖Парсер│   │👤Мария│   │👤Денис│
 │🤖Нетвр│   │🤖Обзвон│   │🤖Напом│   │🤖Монит│
 │👤Мих. │   │       │   │       │   │      │
 └───────┘   └───────┘   └───────┘   └───────┘
```

- Уровень 0: корень (CEO/основатель) — центральная карточка с градиентом
- Уровень 1: отделы — блоки с названием и зоной ответственности
- Внутри отдела: карточки сотрудников (живые 👤 / ИИ-агенты 🤖)
- Соединительные линии через CSS-псевдоэлементы
- При большом количестве отделов — горизонтальный скролл

### 2.2 Карточка сотрудника на схеме

Минималистичная, информация:

```
┌─────────────────────────────────┐
│ 👤 Анна              [живой]    │
│ Руководитель продаж            │
│ +7 988 *** **12                │
└─────────────────────────────────┘
```

- 👤 или 🤖 — иконка типа
- Имя + бейдж «живой» / «ИИ»
- Должность (role) — под именем
- Привязанный аккаунт / номер — мелким текстом (если есть)
- Ховер: подсветка границы
- Клик: форма редактирования (та же, что сейчас)

### 2.3 Пул ресурсов (авто-импорт)

Новый блок под орг-схемой:

- **📦 Ресурсы (авто-импорт из раздела «Ресурсы»)**
- Список чипсов-аккаунтов: 🟢 привязан / 🟡 непривязан
- Каждый чипс: номер/@username → на какого сотрудника назначен (или «непривязан»)
- Непривязанные можно кликнуть → назначить на должность
- Синхронизация: при загрузке страницы подтягиваются все `accounts` из БД, которые не в `org_members`

### 2.4 Вкладка «👥 Список» (была «Сотрудники»)

Без изменений — таблица со всеми сотрудниками, фильтры, сортировка.

### 2.5 Интерактивность

- **Клик по карточке** на схеме — форма редактирования
- **Клик по «непривязан» аккаунту** — предложить привязать к новой должности
- **Кнопки:** Добавить отдел, Добавить сотрудника, Синхронизировать ресурсы, Переключить на таблицу
- Добавление/редактирование — те же формы, что сейчас, просто вызываются кликом по схеме

## 3. Изменения в коде

### 3.1 `axiom/web/index.html` (фронт)

- **Новый CSS** (~100 строк) — для org-диаграммы:
  - `.org-tree` — контейнер с flex-деревом
  - `.dept` — блок отдела с `::before`-линией
  - `.m-card` — карточка сотрудника
  - `.org-root` — корневая карточка CEO
  - `.pool` — блок пула ресурсов
- **Новая функция `renderOrgChart()`** — вместо `renderOrgChartTab()`:
  - Строит HTML-дерево из `data.departments` (рекурсивно)
  - Для каждого уровня отрисовывает карточки
  - Внизу — пул ресурсов
- **Доработка `renderOrgMembers()`** — без изменений
- **Новая функция `renderResourcePool()`** — получает непривязанные аккаунты
- **Новый эндпоинт на фронте** — `api("/api/org/unlinked-accounts")`
- Переименование вкладок: «Схема» → «🗺️ Схема», «Сотрудники» → «👥 Список»

### 3.2 `axiom/web/app.py` (бекенд)

- **Новый эндпоинт `GET /api/org/unlinked-accounts`** — возвращает аккаунты, не привязанные к `org_members`
- **Доработка `GET /api/org/tree`** — в ответ уже включены `department_id` у member'ов, достаточно
- Без изменения БД — таблицы `accounts`, `departments`, `org_members` уже есть

### 3.3 База данных

Изменений в схеме БД не требуется. Таблицы уже есть:
- `org_members` — с полями `kind`, `account_id`, `name`, `role`
- `accounts` — аккаунты/номера
- `departments` — отделы с `parent_id`
