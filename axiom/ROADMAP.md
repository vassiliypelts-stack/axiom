# AXIOM — Роадмап доработок

Зафиксировано из брейншторма. Сгруппировано по волнам. Помечен размер: 🟢 быстро · 🟡 средне · 🔴 крупно.

## Волна A — Структура интерфейса 🟢
- [ ] Сгруппировать левое меню по блокам: **Дашборд / Команда / CRM / Парсинг / Настройки**.
- [ ] Убрать воронку из Дашборда (лишнее там).
- [ ] Объединить **Проекты + Кампании** в один блок.
- [ ] Раздел «Мои агенты» → «Команда». В карточке агента порядок: **телефон → ник в TG → статус подключения (TG активен? WA активен?)**.
- [ ] **Календарь** — отдельная вкладка после «Команда»: видеть назначенные встречи (КЭВ). Сейчас из нашей базы (deals.meeting_at); позже — синхронизация с Google/Яндекс Календарём.
- [ ] **Перетаскивание пунктов меню** вверх/вниз по приоритету (как в Битрикс), порядок сохраняется.

## Волна B — CRM как в Битрикс 🔴
- [ ] **Несколько воронок** (по продуктам). Каждый продукт = своя воронка.
- [ ] Лид «носится» по продукту: при заполнении карточки указывается, какой продукт/воронка.
- [ ] **Карточка лида как в Битрикс** (полноценная: данные, продукт, стадия, история, КП, задачи).
- [ ] **Люди и Компании — раздельно** (как в Битрикс: контакты-люди и компании отдельно, связь между ними).
- [x] В кампании — **прикрепить КП** (PDF-файл), агент отправляет файлом. *(Telegram: агент шлёт файлом по флагу send_kp. WhatsApp-отправку файла — дожать в Baileys.)*

## Волна C — Парсинг и лидген из чатов 🔴
- [ ] **Отдельный раздел «Чаты» (каталог + анализ) — своя база чатов**, не мешать с CRM-контактами.
- [ ] Рабочие аккаунты **неспешно вступают (20-30/день, антибан)**, считают участников, смотрят активность.
- [ ] **Админы чата — отдельной таблицей/окном**, с указанием аккаунта-админа (часто их 2-3 на чат).
- [ ] В карточке агента — **в каких чатах состоит**: название, ссылка, кол-во участников; общая аудитория агента; свод по каждому.
- [ ] Загрузка списков (чатов/ссылок) в парсер.
- [ ] Слушатели чатов **заходят в чаты раз в день**, ищут по ключевым словам запросы (горячие лиды).
- [ ] Найденное → **на обзор оператору**: вижу переписку, решаю — брать ли в лиды → перенос в CRM.
- [ ] Агент **пишет в личку** тем, кто активно пишет в чатах.
- [ ] По каждому чату/каналу — **написать админу(ам)**, показать кто админ.

## Волна D — Контент-постинг в чаты 🟡
- [ ] Отдельное задание агенту: **постить в чаты по расписанию** (текст + картинка/видео), напр. раз в неделю. Объяснялки/прогрев аудитории.

## Волна E — Прогрев + синхронизация 🟡
- [ ] Отдельная настройка прогрева новых аккаунтов: **готовый гармоничный сценарий на 10-15 дней** (по дням, плавно).
- [ ] **Синхронизация общения агентов** в личке (чтобы агенты не дублировали друг друга / не палились).

## Волна E2 — Анти-бан и человекоподобное поведение 🟡
- [x] **Самопроверка через @SpamBot** — аккаунт спрашивает @SpamBot про ограничения/бан, статус пишется в карточку.
- [ ] **Добавлять контакт в записную книжку** аккаунта ПЕРЕД первым сообщением (меньше шансов на спам-флаг).
- [ ] **Человекоподобное поведение** в меру: смотреть сторис ЦА, ставить лайки/реакции, заходить онлайн.
- [ ] **Постинг сторис/картинок по графику** — разнообразно/спонтанно (не по ровному расписанию), для «живости» аккаунта.

## Волна C2 — Свои каналы и вовлечение 🟡
- [ ] Аккаунты **читают свои каналы/чаты** и **отвечают на комментарии** людей, вовлекая в КЭВ.
- [ ] **WA-ярлыки**: помечать лиды ярлыками в WhatsApp (Baileys labels), синхронизировать с тегами/воронкой CRM, чтобы люди были по группам.
